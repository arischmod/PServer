-remftr
template: ster?clnt=<name|pass>&com=remftr&str=str_pattern[&ftr=ftr_pattern&ftr=ftr_pattern...]
          Order of query params is not important.
pattern : A stereotype pattern can use * to match the last part of a stereotype name.
	  so "ster=some_ster.*" will return all stereotypes whose names begin with
	  some_ster. while "ster=*" will return results from all stereotypes.
	  Same principal applies to feature pattern.
descript: Removes any and all (stereotype, feature) pairs matching both str_pattern
	  and ftr_pattern. If no ftr is defined * will be assumed.
example : ster?clnt=name|pass&com=remftr&str=*&ftr=testFtr1
	  ster?clnt=name|pass&com=remftr&str=testStr1.*&ftr=testFtrs5.*
	  ster?clnt=name|pass&com=remftr&str=testStr1.*
returns : 200 OK, 401 (fail, request error), 501 (fail, server error)
