-remstr
template: ster?clnt=<name|pass>&com=remstr&str=<str_pattern>
          Order of query params is not important.
pattern : for stereotype: * | name, name can be a path expression (group1.*)
descript: Removes all stereotypes matching str_pattern
example : ster?clnt=name|pass&com=remstr&str=*
	  ster?clnt=name|pass&com=remstr&str=test.*
returns : 200 OK, 401 (fail, request error), 501 (fail, server error)
200 OK  : example result body:
	  <result>
	    <row>
	      <num_of_rows>1</num_of_rows>
	    </row>
	  </result>
