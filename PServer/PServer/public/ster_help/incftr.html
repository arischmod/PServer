-incftr
template: ster?clnt=<name|pass>&amp;com=incftr&str=str_name[&ftr1=dgr1&ftr2=dgr2...]
          Order of query params is not important: Stereotype and feature names
	  should not end with *.
descript: Increases the feature's value by the given degree for the defined stereotype. If
	  the stereotype didn't have a value for any of the specified features they will be added.
	  If no ftr is specified nothing will be done.
example : ster?clnt=<name|pass>&amp;com=incftr&str=testStr1&testFtr1=12
returns : 200 OK, 401 (fail, request error), 501 (fail, server error)
200 OK  : in this case the response body is as follows
          <?xml version="1.0"?>
          <?xml-stylesheet type="text/xsl" href="/resp_xsl/rows.xsl"?>
          <result>
          <row><num_of_rows>number of relevant rows</num_of_rows></row>
          </result>
