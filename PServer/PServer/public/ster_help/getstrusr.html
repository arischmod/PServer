-getstrusr
template: ster?clnt=<name|pass>&com=getstrusr&str=<str_pattern>&usr=<usr_pattern>
          Order of query params is not important.
pattern : for stereotype, * | name, where name can be a path expression (category1.*).
	  for the users, * | name, where name can be a path expression (group1.subgroup2.*).
descript: for each stereotype matching <str_pattern>, the users matching <usr_pattern>
          are found and returned
example : ster?clnt=name|pass&com=getstrusr&str=professionals&usr=temp.*
returns : 200 OK, 401 (fail, request error), 501 (fail, server error)
200 OK  : example result body:
	  <result>
	    <row>
	      <str>test2</str><usr>temp.2</usr><dgr>0.3</dgr>
	    </row>
	    <row>
	      <str>test2</str><usr>temp.1</usr><dgr>1.5</dgr>
	    </row>
	    <row>
	      <str>test</str><usr>temp.1</usr><dgr>1</dgr>
	    </row>
	  </result>
