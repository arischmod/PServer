-addstr
template: ster?clnt=<name|pass>&com=getstrusr&str=str_pattern[&usr=usr_pattern]
          Order of query params is not important.
pattern : A stereotype pattern can use * to match the last part of a stereotype name.
	  so "ster=some_ster.*" will return all stereotypes whose names begin with
	  some_ster. while "ster=*" will return results from all stereotypes.
	  Same principal applies to user pattern.
descript: Returns all users matching usr_pattern (if one doesn't exist * is assumed)
	  that belong to any stereotype matching str_pattern in (str, usr, degree)
	  tuples.
example : ster?clnt=<name|pass>&com=getstrusr&str=strCat1.*&usr=temp.*
	  ster?clnt=<name|pass>&com=getstrusr&str=*
returns : 200 OK, 401 (fail, request error), 501 (fail, server error)
