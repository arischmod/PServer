-addstr
template: ster?clnt=<name|pass>&com=getstrftr&str=str_pattern[&ftr=ftr_pattern&srt=srt_method]
          Order of query params is not important. srt_method can be either "desc" for
	  descenting or "asc" for ascenting. If no srt_method is defined or if
	  srt_method doesn't much either "asc" or "desc", "desc" is assumed.
pattern : A stereotype pattern can use * to match the last part of a stereotype name.
	  so "ster=some_ster.*" will return all stereotypes whose names begin with
	  some_ster. while "ster=*" will return results from all stereotypes.
	  Same principal applies to user pattern.
descript: Returns all users matching ftr_pattern (if one doesn't exist * is assumed)
	  that belong to any stereotype matching str_pattern in (str, ftr, value)
	  tuples.
example : ster?clnt=<name|pass>&com=getstrftr&str=strCat1.*&ftr=temp.*
	  ster?clnt=<name|pass>&com=getstrftr&str=*&srt=desc
returns : 200 OK, 401 (fail, request error), 501 (fail, server error)
